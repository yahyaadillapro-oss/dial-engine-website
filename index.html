Native Google Architecture. Enterprise-Grade Performance.
Dial Engine is not just a CRM—it is a fully autonomous, self-managing sales infrastructure. Built on Google Drive, powered by Supabase, and commanded entirely through Discord.

The Manager Hub & KPI Radar

📡 60-Second Background Radar

Total operational oversight. An asynchronous Python task continually scans every agent's terminal. When a lead is marked as "TRANSFER", the system instantly pings your Discord management channel with full property and contact details.

📊 Automated Master Database

Forget manual spreadsheets. The Master Hub acts as the single source of truth, automatically logging agent names, Discord IDs, generated UUIDs, status, and CRM links the moment they are hired.

🏆 Daily Leaderboard Generation

Built-in gamification. The system continuously tallies dials and booked transfers, generating a clean KPI snapshot that can be broadcasted directly to your team's #leaderboard channel with a single command.

SIP Inventory Automation

Scaling a floor requires phone numbers. Dial Engine maintains a live SIP Inventory sheet.


When an agent is hired, the system seamlessly pulls an available BulkVS SIP credential, assigns it to their unique UUID, and DMs them the exact UDP configuration for Linphone. When fired, the SIP is instantly scrubbed and returned to the "Available" pool.

Instant-Deploy Agent Terminals

⚡ Auto-Cloning Architecture

No more manual sharing. Firing the hire command commands the Google Drive API to perfectly clone the master CRM template, rename it, organize it into the Dialer Folder, and grant Writer access to the agent's email.

🏡 Zillow Data Injection

Agents dial with leverage. The terminal provides a live, 15-column matrix including property addresses, cities, and states. Agents can pull estimated values and street views directly inside their Spreadsheet Terminal before the call even connects.

🔐 Zero Extensions Required

Operate under a "Zero-Footprint" philosophy. Because the entire system relies on Google Apps Script and secure API bridging, your agents require zero heavy browser extensions, eliminating the risk of credential scraping or lag.

PostgreSQL Lead Vault

🚦 Concurrent "Skip Locked" Tech

Prevent the nightmare of double-calling. The vault utilizes advanced PostgreSQL locking mechanisms. When 50 agents click "Get Leads" simultaneously, the database instantly reserves strictly unique batches for every single request.

🔀 Dynamic Batch Shuffling

Beat list fatigue instantly. Operation Mix & Match allows admins to sweep all un-transferred leads from every agent's CRM, shuffle the entire database pool, and redeploy completely randomized batches to keep momentum high.

🥞 The "Stacking" Engine

Global deployment made easy. Need to feed the floor? The system finds all active agents and appends a fresh stack of leads to the absolute bottom of their current list, without interrupting their workflow or deleting existing notes.

Discord Command Center

Manage a 50-seat floor globally using purely intuitive slash commands. Everything is routed, authenticated, and executed in seconds.

  • !hire @User [Email] [Name]Clones CRM, assigns SIP, applies roles, and DMs station setup.
  • !fire @UserRevokes access, deletes Google Drive file, and recycles leads.
  • !dispense @User [Amt]Pulls fresh leads from the Supabase vault to a specific agent.
  • !dispense_everyone [Amt]Mass global injection to all active agent CRMs simultaneously.
  • !mix_match [Amt]Recycles the entire floor and redistributes randomized batches.
  • !reset_everyoneGlobal panic button. Wipes all CRMs and returns leads to vault.
  • !leaderboardBroadcasts the Daily Leaderboard rankings to the team channel.
  • !statsSilently DMs management a private snapshot of top active dialers.
  • !check @UserView a specific agent's current KPI stats and active SIP credentials.
  • !transfers [@User]Audit the last 10 successful transfers directly from the database ledger.
  • !vaultInstantly returns an exact count of fresh leads available in the pool.
  • !recycle @UserWipes an underperforming agent's CRM and returns leads to the vault.

Security & Telemetry

🛡️ Fernet AES Encryption

The Dial Engine does not just log data—it locks it down. All internal terminal operations are encrypted using cryptography.fernet and stored locally in a secure `.vault` file, unreadable by unauthorized parties.

🗑️ Total Data Eradication

When an agent's contract is terminated via `!fire`, the system doesn't just revoke access. It forces the Google Drive API to permanently delete their CRM file from existence, ensuring your proprietary data cannot be exported post-termination.

📡 Live Streamed Logs

Complete transparency for administrators. System actions, radar syncs, and command executions are stripped of secure data and streamed asynchronously to a private Discord logging channel in real-time.
